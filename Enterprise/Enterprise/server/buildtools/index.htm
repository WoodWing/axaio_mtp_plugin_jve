Build tools - Enterprise Server

There are some pitfalls editing Enterprise source files. Therefore, some modules are generated. Those tools can be found here. (Note that some test tools have been moved to the Build Test since 8.0.)

Generate PHP from WSDL

Whenever something is changed to the WSDL documents, PHP classes needs to be re-generated. In the first pulldown menu, choose the WSDL you have changed. When introducing new web services (messages / operations) it is advisable to choose the "-all-" entry in the second pulldown menu. When web services are adjusted, the "Request and Response classes" entry needs to be used. When complex types are changed, use the "Data classes" entry.
IMPORTANT:After changing the WSDL, please validate it by running the tools in this folder: server/buildtools/wsdl2java/. This is to avoid errors thrown by 3rd party SOAP parsers, as used by system integrators.

Generate SQL scripts from DB model

Whenever something is changed to the DB model (dbmodel.php), SQL scripts needs to be re-generated. Those scripts take care of clean DB installations and DB migrations (for existing databases).
